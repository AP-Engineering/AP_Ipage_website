MES (APQC) Software implementation (Lamb Weston – Oregon)

Rockwell Automation was brought in to implement an entire plant model, which created increased efficiency in production and cost savings. Working with Rockwell, the model was integrated into the existing plant systems. This involved items like creating and tuning cascading PID loops, OPC communications between different equipment and integrating the model into the existing control architecture

Standalone CL2 controller replacement (CVRD – BC)

A stand-alone PID controller was removed and its functionality replaced by the use of an AB SLC 500 PLC. Items like turbidity and flow needed to be accounted for in the design to ensure successful chlorination levels were reached. Feed forward control was added due to a large dead time, and operator control was added to the RSView32 HMI

Modernization of the Sarasota SCADA system (Sarasota Government – Florida)

A path forward for connecting and modernizing the SCADA system across Sarasota County was developed. The engineering report was presented and funding secured for phase I implementation. This involved items such as creating standards and applying them, managing SCADA technicians, and centralizing services (Wonderware Galaxy, Historian, Change Management Software, License Servers, etc…)

Ongoing Support and Maintenance (Gold River - BC)

AP Engineering worked with the Village to support their aging system until they were ready to implement a new design. This involved correcting items like: Level meters not functioning correctly, Pumps starting and stopping unexpectedly, Added alaming for specific events, fixing HMI objects that were not displaying information correctly, replacing a Chart Recorder with Digital DataLogger, etc..

Receiving Line Controls (Lamb Weston – Washington)

Designed and implemented a new control system for the receiving line of a Lamb Weston plant; which included the use of remote ethernet based pneumatics. This involved writing code in the ControlLogix PLC (in function block, ladder, and structured text), while utilizing Plant-PAX objects and tying those objects to the FactoryTalk SE HMI. FAT testing was completed and the controls were successfully commissioned.

Support and Maintenance (CVRD - BC)

I solved communication issues within the OSI PI Historian environment. An existing system continued to lose data due to poor connectivity between the central historian and the remote nodes. I redesigned the OPC communication pathways, added store-and-forward capabilities, as well as failover to virtually eliminate data loss. After the change, the system experienced downtime with the historian as well as remote node failures and still suffered zero data loss.
